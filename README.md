## 📖 项目简介

本项目是一个多平台小说阅读器，支持 Linux（Ubuntu）、Windows、Docker、网页版等多种平台。用户可以通过此项目阅读小说及短篇/短故事等，享受无广告、高度定制化的阅读体验。

|------|------|------|
| 🐧 Linux (Ubuntu) | ✅ 已支持 | 完整功能支持 |
| 🪟 Windows | ✅ 已支持 | 完整功能支持 |
| 🍎 macOS | ✅ 已支持 | 完整功能支持 |

## ✨ 内置功能

基于目前各大知名网络小说平台的付费功能配置：

### 1. 🚫 无广告阅读
- 无弹窗、无视频、无直播干扰
- 为用户提供优质的阅读氛围
- **PS**：没指望这个项目能够盈利，但如果您能够在下方的赞助链接内给一瓶冰露水的赞助，这将是我们更新项目的动力 💙

  - 字号大小（8px - 72px）
  - 行间距、段间距

- **主题模式**：日间/夜间/护眼模式

- 多角色对话识别
- 可修改配速、下载

### 5. 🔌 MCP & Skill 支持 `待实现`

## 🔧 核心功能模块

### [1] 官方渠道
- **功能**：获取官方排行榜信息
- **特性**：
  - 定期自动更新
  - AI 持续检测排行榜变动
  - 支持多种类型的排行榜（月票榜、推荐票榜、收藏榜等）
- **数据核对**：与第三方获取的小说信息进行比对，确保匹配性
- **示例存储**：番茄小说网页端前 15 章在线看内容，用于比对和展示

### [2] 多元分析
- **触发条件**：当前源均无法获取小说正文信息时
- **搜索方式**：百度、360、必应等多种方式全网搜索
- **AI 介入**：
  - 检测到网址无规则对应解析时，自动生成站规则
  - 确保返回信息的准确性
- **定期检测**：实时监测已储存源的可用性并更新

### [3] 开源项目加持
- **目的**：提供稳定权威的接口备用方案
- **应用场景**：
  - 百度、必应等无法访问时的备用解决方案
  - 日常也用于第三方源

### [4] 主副服务端方案
```
┌─────────────┐         BT/磁力/种子          ┌─────────────┐
│   主服务端   │ ───────────────────────────→ │   分服务端   │
│  (脚本生成)  │                              │ (脚本更新)   │
│             │ ←────── 错误上报 ──────────── │             │
└─────────────┘                               └─────────────┘
       ↓                                              ↓
   AI 解析修复                                   应用修复
```

- **信息传输**：采用磁力/BT/种子方式确保安全性
- **加密机制**：公钥 - 私钥加密防止解密

### [5] 人机验证反拦截系统
- **检测**：自动检测人机验证出现
- **处理流程**：
  1. 截取验证画面
  2. 发送给 AI 视觉模型分析
  3. AI 发送指令完成验证
  4. 在窗口中自动执行

## 🚀 快速开始

### 环境要求

- Python 3.9+
- Node.js 18+ (Web 端)
- Docker 20+ (可选)

### 安装方式

#### 方式一：源码安装

```bash
# 克隆仓库
git clone https://github.com/your-username/novel-read.git
cd novel-read

# 安装依赖
pip install -r requirements.txt

# 启动服务端
python server/main.py

# 启动客户端
python client/main.py
```

#### 方式二：Docker 部署

```bash
# 拉取镜像
docker pull your-username/novel-read:latest

# 运行容器
docker run -d \
  -p 8080:8080 \
  -v ./data:/app/data \
  --name novel-reader \
  your-username/novel-read:latest
```

#### 方式三：桌面客户端

```bash
# Windows/macOS/Linux
# 下载对应平台的发行版
# 详见 Releases 页面
```

### 配置说明

1. **服务端配置** (`config/server.yaml`)
```yaml
server:
  host: "0.0.0.0"
  port: 8080
  database: "./data/novels.db"
  
ai:
  code_model: "your-code-model-api"
  vision_model: "your-vision-model-api"
  
channels:
  official: true
  search: true
  opensource: true
```

2. **客户端配置** (`config/client.yaml`)
```yaml
client:
  theme: "dark"
  font_size: 16
  auto_download: false
  
encryption:
  enabled: true
  key_file: "./keys/client.pem"
```
